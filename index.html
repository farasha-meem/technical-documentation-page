Introduction

The GitHub workflow includes creating a repository on github, working on your project in your local machine and finally updating your GitHub repository with the completed project.

This documentation assumes SSH code is already confugred on GitHub and basic knowledge of CLI commands.

Repositories

A software repository, or “repo” for short, is a storage location for software packages. [1]

For this workflow, we will have a GitHub repository as well as a clone repository on the local machine. We will keep both the repositories updated accordingly with each other.

From GitHub to Local

To create a repository on GitHub, press the "Create Repository" button and get the SSH code in order to clone to the local machine.

On the terminal of your local machine, make sure you are in the directory where you want to clone the repository. Note that this will create a new folder with the repository name in the current directory. To clone your GitHub repository copy the SSH code and paste it in the command:

  • git clone SSH-Code-Here

Navigate to the new folder created with the command:

  • cd Repository-Name command
Stage and Commit

The staging and comitting commands are all done through the terminal.

We will add the changes to staging with the command:

  • git add *

to add all changes and once we are satisfied with a task, we can commit the changes with the command:

  • git commit -m "message"
Push Changes

Once we are done updating our project, we are ready to push the changes into the GitHub repository with the command:

  • git push origin main
